[2605.23901] LLMs as Noisy Channels: A Shannon Perspective on Model Capacity and Scaling Laws

This paper introduces a novel 'Shannon Scaling Law' framework that successfully models and predicts non-monotonic LLM behaviors like overtraining and quantization degradation.
  • Models LLM training as information transmission over a noisy channel, mapping parameters to bandwidth and tokens to signal power.
  • Explains non-monotonic performance degradation (U-shaped curves) when scaling without maintaining a sufficient signal-to-noise ratio (SNR).
  • Successfully predicts unseen 12B Pythia model performance up to 307B tokens (pooled R2=0.847) using training data from models under 6.9B.

MuseTalk Production Deployment in Practice — Docker, GPU Serving, Autoscaling, Cost Optimization, Observability | Tomoda Hinata — SaaS/DX

An exceptionally detailed, practical guide on productionizing the MuseTalk lip-sync model, sharing real-world infrastructure landmines and solutions.
  • Advocates for a resident model with an avatar LRU cache instead of launching direct python scripts per request to avoid startup latency.
  • Implements queue-driven processing with idempotency keys and automatic OOM recovery by dynamically shrinking batch sizes.
  • Uses KEDA for queue-depth-based GPU autoscaling, keeping a minimum of 1 warm replica for low latency while scaling batch workers to zero.
  • Optimizes costs using spot GPUs, fp16 precision, VAD silence-skipping, and avatar reuse.
