Building a LLM in pure Rust, Part 2 - Hello Blog

A personal engineering blog post detailing a highly specific performance bottleneck (1.2-2.0s lag) when building an LLM in Rust using the Burn framework versus Candle.
  • Identifies a 1.2 to 2.0 second CPU-bound lag per training step in Burn 0.21's CubeCL backend due to runtime IR compilation and eager-mode autodiff graph construction.
  • Contrasts Burn's overhead with Candle, which avoids intermediate graph tracing by using direct cudarc calls to cuBLAS and handwritten CUDA kernels.
  • Highlights how host-side framework bookkeeping (buffer management and dispatch overhead) can leave the GPU starved and idle.

[2606.06888] Data-Constrained Language Model Pretraining: Improved Regularization and Scaling Laws

An academic paper addressing the highly relevant, data-constrained LLM pretraining regime with a novel scaling law and regularization technique.
  • Proposes Masked-Input Regularization (MIR), an auxiliary next-token prediction loss on masked inputs that improves validation loss in data-constrained regimes.
  • Introduces SoftQ, a new scaling law that couples model size and data size to better fit performance under repeated data epochs than the additive Chinchilla law.
  • Quantifies MIR's regularization gains as equivalent to roughly 1.3 times as much unique training data.
