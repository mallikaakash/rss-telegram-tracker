Python 3.15's Ultra-Low Overhead Interpreter Profiling Mode – Ken Jin's Blog

It provides a brilliant performance engineering deep-dive into Python 3.15's JIT profiling mechanism, which is highly relevant for backend and performance engineers.
  • CPython 3.15 introduces a dual dispatch table mechanism to swap between normal execution and profiling modes without branching overhead.
  • Instead of doubling the interpreter size, the second table maps all instructions to a single recording/profiling instruction that fans out to the normal interpreter.
  • This technique avoids the 6% slowdown on pyperformance caused by previous tail-calling or computed-goto interpreter profiling approaches.

Agent swarms and the new model economics

It details a highly sophisticated multi-agent architecture and model cost-optimization strategy from Cursor, a top-tier engineering team.
  • Cursor scaled agent swarms to rebuild SQLite in Rust, achieving an 80% test suite pass rate in 4 hours using Grok 4.5.
  • The architecture uses a tree-like decomposition with 'Planner' agents (frontier models) delegating to 'Worker' agents (cheaper, faster models) to prevent context drift.
  • Mixing frontier and inexpensive models produced similar quality to using frontier models exclusively, but at a fraction of the cost.

How the 2026 World Cup affected Internet traffic

It is a high-quality data analysis post from Cloudflare (a target team) demonstrating how to establish baselines and normalize global traffic data.
  • Cloudflare established a 'normal' baseline using the median traffic of the four preceding weeks to smooth out day-to-day noise.
  • To compare high-volume and low-volume countries, they used the log₂ ratio of current to baseline traffic, making increases and decreases symmetric.
  • Matches played during overnight/early-morning hours (midnight to 8 AM local time) caused traffic to more than double compared to baseline.
