Unlocking the Codex harness: how we built the App Server

A high-quality engineering deep-dive from OpenAI explaining the architecture and JSON-RPC protocol of the Codex App Server used to power agentic workflows across multiple clients.
  • The Codex App Server uses a bidirectional JSON-RPC protocol to mirror the terminal user interface (TUI) loop, enabling rich interactions like streaming reasoning and emitting diffs.
  • OpenAI chose a custom JSON-RPC protocol over the Model Context Protocol (MCP) because maintaining MCP semantics for complex IDE integrations like VS Code proved difficult.
  • The architecture allows multiple clients (CLI, VS Code, Xcode, macOS app) to reuse the same underlying agent loop and harness without re-implementing agentic logic.

Improving Hugging Face Training Efficiency Through Packing with Flash Attention 2

Directly addresses LLM training efficiency and performance engineering by introducing boundary-aware sequence packing compatible with Flash Attention 2.
  • Introduces `DataCollatorWithFlattening` to pack instruction-tuning examples without padding, eliminating redundant computations on padding tokens.
  • Maintains boundary awareness during packing using `flash_attn_varlen_func` to calculate cumulative sequence lengths, preventing undesired cross-example attention.
  • Achieves up to a 2x throughput increase in tokens/second on datasets with high length variance (like FLAN) across models like Llama-2-7B and Mistral-7B.

Introduction to ggml

A foundational deep-dive into ggml, the lightweight C/C++ machine learning library powering popular local LLM inference engines like llama.cpp and ollama.
  • ggml is a minimalist, self-contained C/C++ library (<5 core files, <1MB compiled binary) optimized for memory-efficient Transformer inference on diverse hardware.
  • It supports quantized tensors to drastically reduce memory footprint and improve performance on consumer hardware (Apple Silicon, ARM, x86_64, CUDA).
  • The library operates at a low level, requiring developers to manually manage memory contexts, tensor allocations, and computational graphs.

Putting RL back in RLHF

Directly advances the reader's top priority of RLHF/reasoning RL by introducing RLOO, a highly efficient online RL training algorithm that outperforms DPO and uses less memory than PPO.
  • RLOO (REINFORCE Leave One-Out) is an online RLHF algorithm that requires only 3 model copies (policy, reference, reward) instead of PPO's 4 (no value model).
  • RLOO uses 50-70% less vRAM than PPO and runs 2x to 3x faster, making online RLHF significantly more accessible and less prone to out-of-memory errors.
  • Treats the entire model completion as a single action rather than token-by-token actions, simplifying the implementation while maintaining competitive win rates against PPO.

LAVE: Zero-shot VQA Evaluation on Docmatix with LLMs - Do We Still Need Fine-Tuning?

Discusses the limitations of traditional exact-string-match VQA evaluation metrics in out-of-distribution settings and proposes LLM-based evaluation.
  • Traditional VQA Accuracy relies on exact string matching, which penalizes correct zero-shot answers that differ in format or specificity from human references.
  • Fine-tuning models on specific benchmarks (like DocVQA) often teaches them syntax formatting rather than actual semantic understanding.
  • Proposes LAVE, an LLM-based evaluation method to better align VQA scoring with human judgment without requiring costly human evaluation.
