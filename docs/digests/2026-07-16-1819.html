Let's Build PlanetScale from Scratch: Infrastructure

A high-quality engineering deep-dive into building database branching infrastructure using Ceph and Kubernetes, directly relevant to storage and systems engineering.
  • Homescale implements database branching using an image/container model where branches are writable clones created from immutable snapshots without copying data.
  • It leverages Ceph block storage and Kubernetes to isolate storage from compute, enabling instant, space-efficient database cloning.
  • The architecture uses intermediate read-only snapshots to branch from already writable containers, preventing byte-for-byte copying of large databases.

What building Shippy taught us about building agents

A solid architectural deep-dive into building a reliable, production-grade AI agent for high-stakes domains, detailing the separation of prompt, skills, and config.
  • Shippy's architecture separates the agent into three components: the 'soul' (system prompt), 'skills' (versioned Docker-baked markdown files defining API/tool interactions), and 'config' (runtime settings, LLM choice, and agent harness).
  • The agent uses OpenClaw as its framework and Claude Opus 4.6, treating model and harness swaps as simple runtime config changes rather than code rebuilds.
  • To ensure reliability in high-stakes maritime operations, Shippy's skills output deterministic verification links and data sources alongside natural language answers.

Model Routing Is Simple. Until It Isn’t.

A highly practical systems-engineering deep-dive into the hidden complexities of LLM routing, focusing on context caching, latency, and dynamic task complexity.
  • Sticker price is a misleading metric for routing; Claude Sonnet 4.6 was 50% cheaper than GPT-4.1 on AppWorld because its lower cache-read pricing heavily rewarded agentic context reuse.
  • Task complexity is often invisible at routing time, as simple-looking prompts (e.g., summarizing a contract) can trigger complex multi-step tool use and retrieval during execution.
  • Effective routing must treat model selection as a multi-dimensional systems optimization problem (balancing cache hits, latency, compliance, and dynamic execution state) rather than a simple classification task.

Newer Models, Same Advantage

Discusses domain specialization, SFT, and DPO for OCR, showing how targeted training beats general models, though highly specific to Portuguese OCR.
  • DharmaOCR outperformed Mistral OCR4 on Brazilian Portuguese through a two-stage pipeline: supervised fine-tuning (SFT) for domain alignment, followed by Direct Preference Optimization (DPO).
  • DPO was specifically used to improve model stability and suppress failure modes (like repetitive loops), which reduced inference time and cost in production.
  • The post emphasizes that domain-specific SFT and preference alignment remain highly effective even as larger, general multimodal models proliferate.

Introduces a new benchmark for evaluating voice-to-voice models, relevant for understanding speech model evaluation, though somewhat product-aligned.
