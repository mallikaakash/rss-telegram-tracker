What I Learned from Reimplementing 40 Multi-Agent LLM Papers

Provides highly technical, Twitter-thread-worthy architectural insights and design patterns from reimplementing 40 multi-agent LLM papers.
  • Most multi-agent papers boil down to ~100 lines of control flow consisting of a loop shape, prompt discipline, and an aggregation rule.
  • Many agent strategies silently depend on sample independence (e.g., self-consistency voting), which breaks if agents interact.
  • Dynamic team structures are better modeled using mutable state rather than rigid, immutable workflow graphs.
  • Validating agent outputs at the turn boundary is critical to prevent error propagation through long execution loops.

Using LoRA for Efficient Stable Diffusion Fine-Tuning

Explains the mechanics of LoRA for parameter-efficient fine-tuning, which is highly relevant to the reader's interest in LLM training.
  • LoRA freezes pre-trained model weights and injects trainable rank-decomposition matrices into attention blocks, drastically reducing VRAM requirements.
  • Applying LoRA to Stable Diffusion's cross-attention layers allows full fine-tuning on consumer GPUs with as little as 11 GB of VRAM.
  • The technique enables faster training and significantly smaller checkpoint file sizes compared to full model fine-tuning.

Probabilistic Time Series Forecasting with 🤗 Transformers

Covers probabilistic time series forecasting, which is highly relevant to the reader's work on industrial sensor data and process optimization.
  • Probabilistic forecasting models the parameters of a distribution (e.g., Student-t) to output prediction intervals and uncertainty, rather than single points.
  • The Time Series Transformer trains a global model across multiple time series to learn shared latent representations.
  • The architecture leverages the vanilla Transformer to handle univariate probabilistic forecasting tasks efficiently.
