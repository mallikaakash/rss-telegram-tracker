Separating signal from noise in coding evaluations

This is a highly technical audit of SWE-Bench Pro revealing critical flaws in LLM coding evaluations, crucial for understanding model benchmarking.
  • An audit by OpenAI reveals that approximately 30% of tasks in the widely used SWE-Bench Pro benchmark are broken or flawed.
  • Key issues identified include overly strict tests (27.4% flagged by pipeline), underspecified prompts, low-coverage tests, and misleading prompts.
  • The findings highlight the extreme difficulty of curating reliable, long-horizon agentic benchmarks and suggest using LLM agents for scalable data quality checks.

Welcome Inkling by Thinking Machines

This post is a deep-dive into a massive 1T open multimodal MoE model featuring novel relative attention, hybrid attention, and speculative multi-token prediction.
  • Inkling is a 975B parameter (41B active) sparse MoE model with 256 experts, supporting a 1M context window across text, image, and audio.
  • Instead of RoPE, it uses a novel relative attention mechanism that learns positions directly in attention logits, combined with a 5:1 hybrid sliding-window to global attention pattern.
  • It supports speculative Multi-Token Prediction (MTP) drafters for faster inference and has day-0 integration with SGLang, vLLM, and llama.cpp.

DiScoFormer: One transformer for density and score, across distributions

This post introduces a novel transformer architecture for joint density and score estimation, relevant to generative modeling foundations.
  • DiScoFormer uses a shared transformer backbone with dual output heads to estimate both density and score in a single forward pass.
  • It leverages a label-free consistency loss (matching the score head to the gradient of the log-density head) to adapt to out-of-distribution data at inference time.
  • The model bypasses the dimensionality limitations of Kernel Density Estimation (KDE) and the retraining requirements of traditional neural score-matching.

Experimenting with the proposed Cross-Origin Storage API in Transformers.js

This article explores a new browser API to solve the massive model caching and duplication issues in client-side WebGPU/Wasm inference.
  • Currently, client-side ML libraries like Transformers.js must re-download and cache identical models (e.g., Whisper-tiny) for every unique origin due to browser cache isolation.
  • The proposed Cross-Origin Storage API allows secure, shared access to cached model and Wasm runtime resources across different websites.
  • This API significantly reduces redundant network bandwidth and client-side storage overhead (saving hundreds of megabytes per user).
