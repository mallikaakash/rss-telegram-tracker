Xet is on the Hub

High-quality ML infrastructure deep-dive explaining how Hugging Face is replacing Git LFS with content-defined chunking to optimize massive model/dataset storage.
  • Replaces Git LFS (which deduplicates at the file level) with Xet storage, utilizing Content-Defined Chunking (CDC) at a ~64KB byte-level.
  • Solves the 're-upload penalty' where a tiny edit to a multi-gigabyte file (like GGUF or SQLite) previously required re-uploading the entire file.
  • Successfully migrated 6% of the Hugging Face Hub's download traffic to Xet infrastructure, validating real-world scale and reliability.

Learning a few things about running SQLite

Practical backend engineering lessons on SQLite query optimization and write-lock timeouts during large operations.
  • Running ANALYZE is critical for SQLite's query planner; it reduced a slow FTS5 query from 5 seconds to 0.05 seconds by generating table statistics.
  • Long-running DELETE operations can lock the database, causing other workers to time out (5-second default) and crash the VM.
  • Mitigated write-lock timeouts by batching cleanup operations into smaller, faster-running chunks.
